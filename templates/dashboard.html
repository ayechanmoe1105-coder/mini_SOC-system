Log Search & Filtering

How to check your logs
  • Place log files in the project logs folder (next to working_app.py). To use another folder, set LOG_DIRECTORY in .env and restart the server.
  • Use .log files (e.g. apache_access.log, ssh_auth.log, firewall.log). The app ingests them on startup and keeps watching for new lines while it runs.
  • Below, enter text, source IP, and/or dates, then click Search to query what is stored in the database. Summary counts appear in the header stats above.
  • Keep the SOC server running and use the dashboard URL shown in the console when you start working_app.py (often http://127.0.0.1:15500 unless you changed SOC_PORT).

Apriori — Association Rules

SOC mode: Apriori uses UTC hour buckets first; if each hour has only one threat_type, it automatically retries with UTC calendar day buckets so types on the same day can co-occur. Optional API: bucket=hour|day|auto, hours lookback. CICIDS mode: each transaction = one flow row with quantile bins (lo/mid/hi) on flow features + outcome_attack / outcome_normal — same dataset family as the Random Forest, but categorical for Apriori. Requires mlxtend for both.
